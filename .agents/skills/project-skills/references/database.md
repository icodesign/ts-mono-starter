# CozyDevs Database Reference

Use this reference when changing Drizzle schema, migrations, database clients, database package exports, or scripts that mutate user/admin state.

## Ownership

- `packages/database` owns database schema, Drizzle configuration, migrations, and database factories.
- `packages/database/src/auth-schema.ts` is generated by Better Auth and should not be hand-edited for normal auth model changes.
- `packages/database/src/schema.ts` is the app-facing schema barrel. It currently re-exports the generated auth schema.
- `packages/database/src/client.ts` owns the Postgres client and Drizzle factory.
- `apps/web` consumes the database through `@cozydevs/database`; do not create independent schema or Drizzle factories in app code.

## Client Rules

- Use `createDatabase(databaseUrl)` for real Postgres connections.
- Use `createMockDatabase()` only for CLI-safe or generation-time paths that need a Better Auth adapter without a live database.
- Do not create raw `postgres(...)` clients outside `packages/database/src/client.ts` unless a new package-level factory is deliberately added.
- In Cloudflare runtime code, use the Hyperdrive connection string from the generated `Env` binding and pass it through the project factories.

## Schema And Migrations

- For app-owned tables, add Drizzle table definitions under `packages/database/src`.
- Keep `packages/database/src/schema.ts` as the stable export surface for consumers.
- When Better Auth config/plugins change model fields, regenerate `packages/database/src/auth-schema.ts` instead of editing it manually.
- Commit generated migrations under `packages/database/drizzle` when schema changes need a durable migration.

## Commands

Generate Better Auth schema after auth model/plugin changes:

```sh
bun run auth:schema
```

Generate and apply Drizzle migrations from the database package:

```sh
bun run --cwd packages/database db:generate
bun run --cwd packages/database db:migrate
```

Use `db:push` only for local development when an explicit migration is not the goal.

```sh
bun run --cwd packages/database db:push
```

Set a user as admin through the database package script:

```sh
bun run --cwd packages/database user:admin <email>
```

## Validation

After database package changes, run:

```sh
bun run --cwd packages/database check
bun run --cwd apps/web check
```

If auth schema generation or migrations changed files, also run:

```sh
bun run fmt:check
```
